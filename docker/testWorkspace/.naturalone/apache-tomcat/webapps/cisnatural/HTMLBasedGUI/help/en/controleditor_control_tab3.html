How to use
You use this tab when developing macro controls. A macro control typically "just" combines several existing controls to form a more complex control with a certain dedicated task. You take what is available and group it into certain units. You use the text area on the left to describe your control directly in XML. For macro controls, you do not need to provide a tag handler class.

Example:

<coltable0 width="$width$">
   <tr>
      <label name="$name$" width="100%" asplaintext="true">
      </label>
   </tr>
   <tr>
      <field valueprop="$valueprop$" popupmethod="openIdValueCombo" width="100%">
      </field>
   </tr>
</coltable0>

The rendered control consists of two lines. The first line displays a label, the second line displays an input field. The two lines are placed into a column container. You see that the attribute WIDTH of the column container and the attribute VALUEPROP of the FIELD are not set in a static way. They refer to the attributes WIDTH and VALUEPROP of the macro control. Remember that you have defined the attributes of the macro control on the Attributes tab.

For a more complex example, have a look at the standard REPORT control. This is a macro control itself. To do so, open editor extension file editor_report.xml.
